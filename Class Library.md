# Class Library

Class Library ဆိုတာကတော့ ကြိုတင်ရေးသားထားတဲ့ class တွေ၊ method တွေ၊ function တွေနဲ့ အခြားအသုံးဝင်တဲ့ code တွေကို စုစည်းထားတဲ့ အစုအဝေးတစ်ခုပဲ ဖြစ်ပါတယ်ခင်ဗျာ။ ကျွန်တော်တို့ရဲ့ project တွေထဲမှာ ဒီ library ကို ထည့်သွင်းပြီးတာနဲ့ အဲ့ဒီထဲမှာပါတဲ့ code တွေကို အလွယ်တကူ ပြန်လည်အသုံးပြုနိုင်မှာ ဖြစ်ပါတယ်။ ဒါကြောင့် code တွေကို ထပ်ခါထပ်ခါ ရေးနေစရာမလိုတော့ဘဲ အချိန်ကုန်သက်သာစေပြီး project တစ်ခုလုံးရဲ့ တည်ဆောက်ပုံကိုလည်း ပိုမိုကောင်းမွန်စေပါတယ်။

### Core Concepts of a Class Library

- **Code Reusability (Code များကို ပြန်လည်အသုံးပြုနိုင်ခြင်း):** Class Library ရဲ့ အဓိက အားသာချက်ကတော့ code တွေကို တစ်ခါရေးရုံနဲ့ project တိုင်းမှာ ပြန်လည်အသုံးပြုနိုင်တာပါပဲ။ ဥပမာအနေနဲ့ ပြောရရင်၊ ကျွန်တော်တို့ရဲ့ application တစ်ခုမှာ email ပို့တဲ့ function တစ်ခုလိုတယ်ဆိုပါစို့။ အဲ့ဒီ function ကို class library တစ်ခုအနေနဲ့ တည်ဆောက်ထားလိုက်ရင် နောက်ပိုင်းတွေမှာ email ပို့ဖို့လိုအပ်တဲ့ တခြား project တွေမှာလည်း ဒီ library ကိုပဲ ပြန်သုံးလိုက်ရုံပါပဲ။ အစကနေပြန်ရေးစရာ မလိုတော့ပါဘူး။
- **Encapsulation (အချက်အလက်များကို လုံခြုံစွာသိမ်းဆည်းခြင်း):** Class Library တွေက ရှုပ်ထွေးတဲ့ လုပ်ဆောင်ချက်တွေကို အပြင်ကနေ တိုက်ရိုက်ထိန်းချုပ်လို့မရအောင် ဖုံးကွယ်ထားပေးနိုင်ပါတယ်။ ကျွန်တော်တို့က library ထဲက class ဒါမှမဟုတ် method ကို ခေါ်သုံးလိုက်ရုံနဲ့ သူဘာအလုပ်လုပ်သွားလဲဆိုတာကိုပဲ သိရင်ရပြီး အတွင်းပိုင်းမှာ ဘယ်လိုအလုပ်လုပ်လဲဆိုတာကို အသေးစိတ်သိနေစရာမလိုပါဘူး။ ဒါက ကျွန်တော်တို့ရဲ့ project ကို ပိုပြီး စနစ်ကျစေပါတယ်။
- **Modularity (အစိတ်အပိုင်းများအလိုက် စုစည်းနိုင်ခြင်း):** Project တစ်ခုကို အပိုင်းလေးတွေခွဲပြီး တည်ဆောက်တဲ့နေရာမှာ Class Library တွေက အရမ်းအသုံးဝင်ပါတယ်။ လုပ်ဆောင်ချက်အလိုက် library တွေခွဲထားခြင်းအားဖြင့် project ကို နားလည်ရလွယ်ကူစေပြီး ပြုပြင်ထိန်းသိမ်းရတာလည်း ပိုလွယ်ကူစေပါတယ်ခင်ဗျာ။

### How a Class Library Works

Class Library တစ်ခု ဘယ်လိုအလုပ်လုပ်လဲဆိုတာကို ဥပမာလေးနဲ့ ရေးပြပါမယ်။

ကျွန်တော်တို့မှာ သင်္ချာဆိုင်ရာ တွက်ချက်မှုတွေလုပ်ဖို့ `Calculator` ဆိုတဲ့ Class Library တစ်ခုရှိတယ်ဆိုပါစို့။ အဲ့ဒီ library ထဲမှာ `Add` (ပေါင်းခြင်း), `Subtract` (နှုတ်ခြင်း), `Multiply` (မြှောက်ခြင်း) နဲ့ `Divide` (စားခြင်း) ဆိုတဲ့ method လေးတွေ ပါပါမယ်။

```csharp
// ကျွန်တော်တို့ရဲ့ Calculator.dll (Class Library) ထဲမှာရှိမယ့် Code
namespace OurMathLibrary
{
    public class Calculator
    {
        public int Add(int a, int b)
        {
            return a + b;
        }

        public int Subtract(int a, int b)
        {
            return a - b;
        }
    }
}
```

အခု ကျွန်တော်တို့ရဲ့ main application ထဲကနေ အပေါ်က `Calculator` library ကို သုံးချင်တယ်ဆိုရင် အောက်ကလိုမျိုး ရေးရုံပါပဲ။

```csharp
// ကျွန်တော်တို့ရဲ့ Main Application
using OurMathLibrary; // ကျွန်တော်တို့ရဲ့ Class Library ကို အသုံးပြုရန် ကြေညာခြင်း

class Program
{
    static void Main(string[] args)
    {
        Calculator calculator = new Calculator();
        int result = calculator.Add(5, 3);
        Console.WriteLine("The result is: " + result); // Output: The result is: 8
    }
}
```

ဒီမှာဆိုရင် ကျွန်တော်တို့ဟာ `Add` method ရဲ့ အတွင်းပိုင်း code ကို ပြန်ရေးစရာမလိုဘဲ `Calculator` class library ကို ခေါ်သုံးလိုက်ရုံနဲ့ အလုပ်ဖြစ်သွားတာကို တွေ့ရမှာပါခင်ဗျာ။

### Class Library vs. Framework

Class Library နဲ့ Framework က ဆင်တူသလိုထင်ရပေမယ့် အဓိကကွာခြားချက်တစ်ခုရှိပါတယ်။

- **Class Library:** ကျွန်တော်တို့ရဲ့ code က Class Library ကို ခေါ်သုံးတာဖြစ်ပါတယ်။ ဘယ်အချိန်မှာ ဘယ် library ကို ခေါ်သုံးမလဲဆိုတာကို ကျွန်တော်တို့က ထိန်းချုပ်ပါတယ်။ ("You call the library.")
- **Framework:** Framework က ကျွန်တော်တို့ရဲ့ code ကို ခေါ်သုံးတာဖြစ်ပါတယ်။ Framework က project တစ်ခုလုံးရဲ့ အလုပ်လုပ်ပုံစီးဆင်းမှုကို (flow) ကို ထိန်းချုပ်ထားပြီး လိုအပ်တဲ့နေရာတွေမှာ ကျွန်တော်တို့ရေးထားတဲ့ code ကို သူက လှမ်းခေါ်သုံးတာဖြစ်ပါတယ်။ ("The framework calls your code.")

Library ဆိုတာက ကိရိယာအစုံပါတဲ့ သေတ္တာတစ်လုံးနဲ့တူပြီး၊ framework ဆိုတာကတော့ အိမ်တစ်လုံးရဲ့ အခြေခံအုတ်မြစ်နဲ့ တည်ဆောက်ပုံတစ်ခုလုံးနဲ့ တူပါတယ်ခင်ဗျာ။ ကျွန်တော်တို့က အဲ့ဒီအုတ်မြစ်ပေါ်မှာ လိုအပ်တာတွေကို ဆက်ပြီးဖြည့်စွက်တည်ဆောက်ရတာပါ။
